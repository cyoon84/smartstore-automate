# 우체국택배 전체 발송 루틴

이 스킬은 두 단계로 나눠진 한 번의 흐름이다. 단계 사이에 사용자가 우체국택배 사이트에 직접 업로드하는 시간이 필요하므로, 첫 단계 결과를 보고한 뒤 사용자가 송장 파일을 가져올 때까지 기다린다.

## 흐름 개요

```
[1단계 변환]  스마트스토어 출고 엑셀  →  우체국택배 업로드용 .xls
       ↓
   사용자가 우체국택배 사이트에 업로드 → 송장 엑셀 다운로드 (대기)
       ↓
[2단계 매칭]  스마트스토어 + 우체국 송장  →  발송처리 엑셀
       ↓
[3단계 알림]  #all-finchmart Slack 채널에 처리 건수 메시지
```

## 1단계 — 변환

사용자에게 두 가지를 묻는다 (인자로 같이 들어오면 그 값을 사용):
- 스마트스토어 출고 엑셀 파일 경로
- 비밀번호 (없으면 빈 값으로)

물은 다음 `scripts/convert.py`를 실행한다:

```bash
pip3 install msoffcrypto-tool pandas openpyxl xlrd xlwt -q
python3 ~/.claude/skills/epost-flow/scripts/convert.py "<스마트스토어_경로>" "<비밀번호>"
```

스크립트 표준출력 형식:
- `✅ 변환 완료: <output_path> (N건)` — 정상 종료

### 1단계 종료 보고

변환이 끝나면 다음 형식으로 사용자에게 보고하고 **여기서 멈춘다**:

```
✅ 1단계 변환 완료
파일: ~/smartstore-project/output/우체국택배/우체국택배_업로드용_YYYYMMDD.xls
변환 건수: N건

이제 우체국택배 사이트에 업로드해 주세요.
업로드 후 받은 송장 엑셀(등기번호 포함)을 첨부하거나 경로를 알려주시면 2단계를 진행합니다.
```

사용자의 다음 메시지에서 송장 파일이 올 때까지 능동적으로 다음 단계로 넘어가지 않는다.

## 2단계 — 송장 매칭

사용자가 우체국 송장 파일(예: `2026050656246453_20260506.xls`)을 첨부하거나 경로를 주면, 이전에 받았던 스마트스토어 경로/비밀번호를 그대로 재사용해서 `scripts/match_invoices.py`를 실행한다:

```bash
python3 ~/.claude/skills/epost-flow/scripts/match_invoices.py "<스마트스토어_경로>" "<비밀번호>" "<우체국송장_경로>"
```

표준출력 형식:
- `DONE:<output_path>:<filled_count>` — 정상 종료, 매칭된 건수
- `RECIPIENTS:이름1|이름2|...` — 매칭된 수취인 이름 (중복 제거)
- `UNMATCHED:이름1(전화1)|이름2(전화2)...` — 매칭 안 된 수취인이 있을 때

미매칭은 보통 다른 날 주문이 섞여 있는 경우이므로 참고용으로만 안내한다.

## 3단계 — Slack 알림

매칭이 끝나면 `#all-finchmart` 채널에 메시지를 보낸다. `slack_send_message` 도구를 사용. **수취인 이름 명단을 반드시 포함**한다:

```
channel: all-finchmart  (또는 채널 ID)
text:
우체국택배 발송처리 완료 ✅
• 변환: {convert_count}건
• 송장 매칭: {filled_count}건 — 이름1, 이름2, 이름3
• 미매칭: {unmatched_count}건 (다른 날 주문 가능성)
• 결과: 스마트스토어_발송처리_YYYYMMDD-우체국.xlsx
```

수취인 이름은 `RECIPIENTS:` 라인을 `|` 로 split 해서 콤마+공백으로 join 한 결과를 사용한다.

채널을 이름으로 바로 찾지 못하면 `slack_search_channels` 로 `all-finchmart`를 검색해 채널 ID를 얻은 뒤 보낸다.

## 마무리 보고

사용자에게 짧게 정리:

```
✅ 우체국플로우 완료
• 변환: {convert_count}건
• 매칭: {filled_count}건 (미매칭 {unmatched_count}건)
• 발송처리 파일: ~/smartstore-project/output/발송처리/스마트스토어_발송처리_YYYYMMDD.xlsx
• Slack #all-finchmart 알림 전송 완료
```

## 참고

- 변환 결과는 `.xls` (xlwt) 형식으로 저장됨 — 우체국택배 사이트는 .xlsx를 받지 않음.
- 변환 시 주문번호 기준으로 중복 제거 (한 주문 = 한 행).
- 고정값: 중량 3kg, 부피 80, 내용품코드 "생활용품", 분할접수 N.
- 매칭 기준: 수취인명(스마트스토어) ↔ 받는분(우체국), 수취인연락처1 ↔ 받는분 전화번호 (하이픈 제거 후 비교).
- 송장번호 컬럼에 우체국 `등기번호` 값을 입력.
- 이 스킬은 변환과 매칭을 **별도 단계로** 처리해야 한다. 두 단계 사이에는 반드시 사용자의 우체국 사이트 업로드 시간이 필요하므로, 1단계 끝에서 멈추고 사용자 입력을 기다린다.
